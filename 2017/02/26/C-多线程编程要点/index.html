改善 std::thread

　　在工程实践中，你会发现std::thread并不像想象中的那么完美。一个明显的问题是，当std::thread即将析构时，你要人为地确保实际的线程已经结束运行了或者已经分离了(可以分别调用它的join()和detach()成员函数)，否则程序将会抛出std::terminate异常。当然，由于std::thread的析构函数没有保证正确地释放资源，这也同时导致了std::thread不是异常安全的。
